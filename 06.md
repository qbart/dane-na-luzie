# AI jako wsparcie analityczne (17:00 - 17:30)

AI potrafi znacząco przyspieszyć pracę z danymi - zarówno w obszarze pisania zapytań SQL, jak i interpretowania wyników.
Nie zastępuje zrozumienia modelu danych, ale może być świetnym narzędziem, gdy wiemy, jak go użyć rozsądnie.

W tym rozdziale zobaczysz, jak traktować AI jako asystenta analitycznego, który:
- pomaga budować zapytania,
- podpowiada, jakie pytania warto zadać,
- wyjaśnia wyniki,
- sugeruje kolejne kroki analityczne.

Celem jest nauczyć się, jak **wykorzystać AI świadomie, a nie ślepo kopiować jego odpowiedzi**.


## Jak AI może wspierać w pisaniu SQL

AI dobrze radzi sobie z:

- tworzeniem bazowych zapytań,
- refaktoryzacją długich zapytań,
- tłumaczeniem zapytania na prosty język,
- generowaniem kilku wariantów rozwiązania.

Przykładowy prompt:

> Mam taki model danych:
> 
> CREATE TABLE student (
>    id    BIGSERIAL PRIMARY KEY,
>    name  TEXT      NOT NULL
> )
>
> CREATE TABLE course (
>    id   SERIAL PRIMARY KEY,
>     name TEXT   NOT NULL
> );
> 
> CREATE TABLE lesson (
>     id        SERIAL  PRIMARY KEY,
>     course_id INTEGER NOT NULL REFERENCES course(id),
>     name      TEXT    NOT NULL
> );
> 
> CREATE TABLE enrollment (
>     id          SERIAL      PRIMARY KEY,
>     student_id  INTEGER     NOT NULL REFERENCES student(id),
>     course_id   INTEGER     NOT NULL REFERENCES course(id),
>     enrolled_at TIMESTAMPTZ DEFAULT now()
> );
> 
> Napisz zapytanie, które zwróci liczbę zapisanych studentów na każdy kurs w porządku malejącym.
> Pokaż SQL i krótko wyjaśnij, jak działa.

Dlaczego warto tak robić:
- wymuszasz, aby AI podało uzasadnienie (łatwiej wykryć błąd),
- określasz model danych — AI działa wtedy precyzyjniej,
- dostajesz odpowiedź, którą możesz zweryfikować.
